Our Approach

Two hard problems — one clear focus

📥

Hard Problem #1 · Smart Ingestion

Employee uploads a PDF resume. GPT-4o extracts skills, proficiency levels, years of experience, and project history into a structured profile. Inferred skills (e.g. Next.js → React) are flagged with confidence scores. Profile lands in an HR review queue before going live.

🔎

Hard Problem #2 · Semantic Search

HR types a natural language query. All approved profiles are summarised and sent to GPT-4o alongside the query. The model returns a ranked list — each result has a match score (0–100) and a plain-English reason. No vector embeddings — reasoning quality is better with a single well-structured prompt.

Architecture

Clean three-tier structure

Frontend
  • React + Vite
  • TanStack Router
  • shadcn/ui + Tailwind
  • Axios (JWT auth)
  • Role-aware UI (HR / Employee)
⇄
Backend (NestJS)
  • AuthModule — tokens + guards
  • ProfilesModule — upload + queue
  • AiModule — GPT-4o calls
  • SearchModule — NL search
  • FilesModule — MinIO
  • Drizzle ORM — PostgreSQL
⇄
Data Layer
  • PostgreSQL 16
  • MinIO (PDF storage)
  • OpenAI GPT-4o API
Auth

Token-based sessions, role guards (HR / Employee), @Global() AuthModule

Schema

users → profiles → profile_skills → skills, projects, sessions

Infra

Single docker-compose.yml spins up Postgres + MinIO with bucket pre-created
